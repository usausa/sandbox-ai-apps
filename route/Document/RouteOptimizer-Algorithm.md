# RoutePlanner 足順生成アルゴリズム（LLM 向け指示書）

これは `RoutePlanner`（route）がローカルで実行している足順生成処理（`RouteOptimizer` と
`TravelTimeMatrixBuilder`）を、LLM がそのまま再現できるように命令形で書き起こしたものです。
あなた（LLM）は、調査員1名・1日分の訪問先一覧から、移動を最小化しつつ時間帯指定・勤務時間・
昼休憩を守る訪問順（足順）を **1つ** 組み立て、各立ち寄りの時刻・移動・待機・違反、全体集計、
未割当を算出してください。以下の手順と設定値に厳密に従ってください。

## 入力

- **訪問先リスト**（各訪問先 = 1件）。各訪問先は次を持つ:
  - `VisitId`（一意）, `CustomerName`, `Address`
  - `Latitude` / `Longitude`（座標）
  - `Category`（`General`=一般 / `Business`=業務）
  - `ServiceMinutes`（個別の調査所要分。未指定または0なら種別の既定値を使う）
  - `WindowStart` / `WindowEnd`（指定時間帯 HH:mm。無い場合あり）
  - `WindowStrict`（`Strict`=厳守 / `Preferred`=希望）
  - `Priority`（`High` / `Medium` / `Low`）
  - `BuildingGroupId`（同一建物＝集合住宅の住戸をまとめる識別子。無い場合あり）
  - `AccessNote` / `HazardNote`（参考情報）
- **共通設定**（下表）。

## 設定値一覧

### 共通設定（CommonSettings）

| 設定キー | 既定値 | 用途 |
|---|---|---|
| `WorkStart` | `09:00` | 勤務開始時刻。`DepartTime` 未設定ならこの時刻に拠点を出発する |
| `WorkEnd` | `18:00` | 勤務終了時刻。これを超える着手は不可（残業許可時は下記まで許容） |
| `DepartTime` | （未設定） | 実際の拠点出発時刻。未設定なら `WorkStart` を使う |
| `LunchStart` | `12:00` | 昼休憩を取り始める基準時刻 |
| `LunchEnd` | `13:00` | 昼休憩終了の目安（参考。実際の消費は `LunchMinutes`） |
| `LunchMinutes` | `60` | 昼休憩の所要分。`0` 以下なら昼休憩を取らない |
| `OfficeLatitude` | `35.6618` | 出発・帰着拠点の緯度 |
| `OfficeLongitude` | `138.5683` | 出発・帰着拠点の経度 |
| `OfficeAddress` | 甲府市役所 | 拠点住所（参考） |
| `ReturnToOffice` | `true` | 最後に拠点へ戻るか |
| `AverageSpeedKmh` | `25` | 平均時速（移動時間の算出に使う） |
| `RoadDetourFactor` | `1.3` | 直線距離 → 道路距離への補正係数 |
| `WeatherDurationFactor` | `1.0` | 天候による移動時間の延長係数 |
| `Weather` | `Sunny` | 天候（`Sunny`/`Cloudy`/`Rain`/`Snow`/`Heat`） |
| `GeneralServiceMinutes` | `20` | 一般の既定調査分 |
| `BusinessServiceMinutes` | `40` | 業務の既定調査分 |
| `BufferMinutes` | `2` | 異なる訪問先へ移る際の準備分 |
| `MaxVisits` | （無制限） | 1日の最大訪問件数 |
| `AllowOvertime` | `false` | 残業を許すか |
| `MaxOvertimeMinutes` | `0` | 許容する残業分（`AllowOvertime` が true のとき有効） |
| `DefaultWindowStrictness` | `Preferred` | 時間帯の既定の厳守区分 |
| `Objective` | `MinimizeTravel` | 最適化方針（総移動の最小化） |
| `TravelMode` | `Car` | 移動手段（参考） |

> 補足: **稼働終了の上限**（`endLimit`）= `WorkEnd` ＋（`AllowOvertime` が true なら `max(0, MaxOvertimeMinutes)`、false なら 0）。
> **実出発時刻**（`EffectiveDepartTime`）= `DepartTime` があればそれ、無ければ `WorkStart`。

### アルゴリズム定数（RouteOptimizer 内の固定値）

| 定数 | 値 | 用途 |
|---|---|---|
| 地域セルの一辺 | `0.0025` 度（約250m四方） | 近接訪問を同一「地域」とみなすグリッドの大きさ |
| 地域結束ペナルティ | `1000.0` | 同一地域の未処理を残して別地域へ移ろうとする候補に課す擬似コスト |
| 枠切迫マージン | `10` 分 | 終了がこの分数以内に迫る時間帯指定を「切迫」とみなす閾値 |
| 距離タイブレーク重み | `0.1` | 移動分が同点のとき、実距離(km)が近い候補を先に選ぶための重み |
| 時間帯遵守ボーナス | `20.0` | 今なら枠内に着手できる時間帯指定候補を優先するための減点 |
| 窓アクティブ先読み | `0` 分 | 開始がこの分数以内に迫った時間帯指定も「開いている」とみなす |
| 局所探索の最大反復 | `20` 回 | 2-opt/Or-opt の最大パス数（改善が無くなれば早期終了） |

## 共通の計算ルール

時刻はすべて「その日の 00:00 からの経過分（整数）」で計算し、最終的に HH:mm に戻してください。

### 2地点間の移動見積り

1. 2点の緯度経度から **ハバーサイン直線距離(km)** を求める（地球半径 6371km）。
2. **道路距離(km)** = 直線距離 × `RoadDetourFactor`。
3. **移動時間(分)** = `道路距離 ÷ AverageSpeedKmh × 60 × WeatherDurationFactor` を **切り上げ**た整数。
4. 出力する距離は道路距離(km)、移動時間はこの整数(分)。

### 調査所要時間（サービス時間）

- 訪問先の `ServiceMinutes` が 1 以上ならその値。
- そうでなければ `Category` が `Business` なら `BusinessServiceMinutes`、`General` なら `GeneralServiceMinutes`。

### 移動バッファ

- 直前の立ち寄りが訪問先で、かつ次の訪問先が **別の建物**（`BuildingGroupId` が一致しない）なら `BufferMinutes` を移動に加算する。
- **同一建物**（`BuildingGroupId` が一致）への移動、および拠点出発の直後は、移動・バッファともに 0。

### 地域キー

- 緯度・経度をそれぞれ `0.0025` 度のグリッドに丸めた (行, 列) を「地域」とする（約250m四方の同一セルを同一地域とみなす）。

### 優先度ペナルティ（スコア用）

- `High` = `-20` / `Medium` = `0` / `Low` = `+10`。

## 手順

### ステップ1: 貪欲法（最近傍）で初期順序を決める

現在地を拠点（`EffectiveDepartTime`・拠点座標）から始め、訪問先を 1 件ずつ選んでいきます。
残り候補が無くなるか、選べる候補が無くなるか、`MaxVisits` に達するまで繰り返してください。

各ステップで:

1. **昼休憩の挿入判定**: まだ昼休憩を取っておらず、現在時刻が `LunchStart` 以上なら、現在時刻に `LunchMinutes` を加えて昼休憩を消費する（その回は訪問選択をせず次の反復へ）。`LunchMinutes` が 0 以下なら昼休憩は取らない。
2. **各候補について着手時刻を見積もる**:
   - `到着 = 現在時刻 + 移動時間 + バッファ`
   - `着手 = max(到着, WindowStart)`（時間帯指定が無ければ到着 = 着手）
   - `早着待ち = 着手 - 到着`
3. **実行不可の候補は除外する**:
   - `着手 > WindowEnd` かつ `WindowStrict = Strict` の候補は選ばない。
   - `着手 + 調査時間 > endLimit`（稼働終了上限超過）の候補は選ばない。
4. **スコアを計算し、最小スコアの候補を選ぶ**（小さいほど優先）:
   ```
   スコア = 着手(分)
          + 移動時間(分) × 0.1
          + 道路距離(km) × 0.1            （距離タイブレーク重み）
          + 優先度ペナルティ
          + （時間帯指定が無ければ +5）
   ```
   - **時間帯遵守ボーナス**: 候補に `WindowEnd` があり、`着手 ≤ WindowEnd` で、かつ（`WindowStart` が無い、または `WindowStart ≤ 現在時刻 + 0分`＝窓が既に開いている）なら、スコアから `20` を引く。
   - **地域結束ペナルティ**: 直前が訪問先で、かつ「現在地域に時間帯指定の無い未訪問が残っている」かつ「終了が10分以内に迫る時間帯指定候補が無い（切迫していない）」かつ「その候補に `WindowEnd` が無い」かつ「その候補が現在地域と別の地域」なら、スコアに `1000` を足す（＝同一地域を片付けてから移動させる）。
5. **同一建物の住戸寄せ**: 選んだ候補が `BuildingGroupId` を持つなら、同じ `BuildingGroupId` かつ同じ時間帯（`WindowStart`/`WindowEnd` が一致）の住戸を `CustomerName` の昇順に並べ、その先頭の住戸を実際の訪問先とする（ただしその住戸の着手＋調査が `endLimit` を超えない場合のみ寄せる）。
6. 選んだ訪問先で現在時刻・現在地・現在の建物・現在地域を更新し、足順に追加する。
7. どの候補も選べなくなったら終了。**残った訪問先はすべて「未割当」** とする。

### ステップ2: 局所探索（2-opt / Or-opt）で行き来（交差）を解消する

初期順序の訪問件数が 3 未満なら、この最適化は行いません。そうでなければ:

- **2-opt**: 足順中の区間 `[i..j]` を反転する並べ替えを試す。
- **Or-opt**: 1 件を別の位置へ移動する並べ替えを試す。
- それぞれの並べ替え後に **ステップ4 の要領でスケジュールを再現** し、実行可能（厳守枠超過・稼働終了超過が無い）であり、かつ次の優先順位で **厳密に改善** する場合のみ採用する:
  1. 時間帯（厳守）違反の件数が少ない
  2. （同点なら）残業が少ない
  3. （同点なら）総移動時間が短い
  4. （同点なら）総移動距離(km)が短い
- 改善が起きる限り繰り返し、改善が無くなるか **20 巡** に達したら終了する。

### ステップ3: 同一建物内の住戸を名称順に整える

最終順序の中で、**連続する** 同一 `BuildingGroupId` かつ同一時間帯の住戸の並びを、`CustomerName` の昇順に整列する（座標・時間帯が同じなので所要時間には影響しない）。

### ステップ4: スケジュールを時系列に確定する

確定した順序に沿って、拠点出発から帰着まで時刻を組み立てます。

1. **出発拠点**（区分=拠点）を `EffectiveDepartTime` に置く。
2. 各訪問先について順に:
   - まだ昼休憩を取っておらず現在時刻が `LunchStart` 以上なら、先に **昼休憩**（区分=昼休憩、`LunchMinutes` 分）を挿入する。
   - 移動を見積もり、`到着 = 現在時刻 + 移動時間 + バッファ`。
   - `着手 = max(到着, WindowStart)`、`待機 = 着手 - 到着`。
   - `着手 > WindowEnd` の場合: `WindowStrict = Strict` ならこの順序は **不成立**（ステップ2では破棄）。`Preferred` なら **時間帯違反フラグ** を立てて続行。
   - `調査終了 = 着手 + 調査時間`。`調査終了 > endLimit` ならこの順序は **不成立**。
   - この訪問の立ち寄り（到着・出発＝調査終了・調査分・直前からの距離/移動分・待機・違反）を記録し、現在時刻 = 調査終了 とする。
3. `ReturnToOffice` が true なら拠点へ戻る移動を加え、**帰着拠点**（区分=拠点）を置く。
4. **残業 = max(0, 帰着（または最終）時刻 − WorkEnd)**。

## 出力

次を算出して返してください:

- **足順（立ち寄りの並び）**。各立ち寄りごとに:
  - 順番、区分（拠点 / 訪問 / 昼休憩）、ID（訪問先は `VisitId`、拠点は `OFFICE_START`・`OFFICE_END`、昼は `LUNCH`）
  - 到着時刻・出発時刻（HH:mm）
  - 調査分、直前からの移動距離(km)・移動時間(分)、待機(分)
  - 時間帯違反フラグ
- **全体集計**:
  - 訪問件数、総移動距離(km, 小数2桁)、総移動時間(分)、総調査時間(分)、総待機時間(分)
  - 稼働開始（＝出発時刻）・稼働終了（＝帰着時刻）、残業(分)、時間帯違反の件数
- **未割当の訪問先一覧**（当日に組み込めなかったもの）。

## 重要な原則

- 時間帯指定（特に `Strict`=厳守）の遵守を、移動効率より優先する。
- 近接する訪問（とくに同一 `BuildingGroupId`）はまとめて回り、「A→C→B」のような行き来や、いったん離れて元の隣接地域へ戻る「地域の分割訪問」を避ける。
- ただし時間帯指定のためにやむを得ず離れる移動は、行き来とみなさない。
- 距離は必ず上記のハバーサイン＋道路係数の式で求め、座標から目分量で推測しない。
